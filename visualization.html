The Vision

🔬

Zoom In

See an individual entity's life story, memories, behavior tree decisions, lineage, and relationships. Watch it hunt, flee, remember, and teach.

🌍

Zoom Out

Watch civilizations rise and fall. See species diverge across biomes, trade routes emerge between settlements, and cultures develop unique signal languages.

📖

Read the Story

Auto-detected narrative arcs — rivalries, alliances, migrations, extinctions — optionally narrated as prose by an LLM. Every event is logged for replay.

Architecture

Three-component system: a Rust simulation engine, Protobuf wire protocol, and TypeScript browser viewer, communicating via WebSocket.

Rust

Simulation Engine

Tick-based, deterministic ECS simulation

  • hecs ECS world wrapper
  • 15-system tick pipeline
  • Behavior tree engine (~300 LOC)
  • Genetic programming / evolution
  • Event sourcing + snapshots
  • Noise-generated terrain
  • Grid spatial hash index
  • Deterministic RNG (ChaCha8)
events
commands
Protobuf

Wire Protocol

Language-neutral shared schema

  • world.proto — state & deltas
  • events.proto — all event types
  • commands.proto — client actions
  • prost (Rust) + ts-proto (TS)
  • WebSocket transport
  • Delta streaming (changes only)
  • Viewport filtering
  • LOD data levels

Server → Client Messages

WorldSnapshot

Full state dump sent on connect or timeline seek

TickDelta

Per-tick changes: spawns, moves, updates, deaths, resource changes

EntityDetail

Full entity details: memory, drives, behavior tree state, lineage

NarrativeUpdate

Story arc events, protagonist tracking, tension curves

Client → Server Messages

SubscribeViewport

Camera position + zoom level — determines what gets streamed

SimulationCommand

Kill, spawn, modify terrain, pause/resume

RequestEntityDetail

Request full state for a selected entity

SetSpeed / SeekTick

Adjust tick rate or jump to a specific historical tick

Tick Pipeline

Each simulation tick executes systems in a fixed, deterministic order. Same seed = same simulation. Every run is reproducible.

Entity Components

Entities are composed of plain data structs. Complexity builds in layers across implementation phases. No methods — all logic lives in systems.
